오늘은 제품에 다국어를 적용하면서 번역 키를 생성하고 업로드하는 과정을 자동화한 스킬을 만들어본 과정을 정리해보겠다.

다국어를 적용할 때 실제로 가장 손이 많이 가는 부분은 번역 자체가 아니라 그 앞단 작업이었다. 
화면에 있는 UI 텍스트를 모두 찾아내고, 규칙에 맞게 키 이름을 짓고, 중복을 확인해서 Lokalise에 올리는 과정이 필요하다. 피처 하나당 텍스트가 수백 개씩 나오는데 이걸 일일이 수작업으로 하려니 너무 막막해서, 이 과정을 자동화하는 파이프라인을 구축해보기로 했다. 

전체 흐름은 간단히 말해서 Figma에서 텍스트를 추출하고, 키 생성 규칙을 적용해 키를 만든 뒤 스크립트로 Lokalise에 업로드하는 방식이다. 

## 번역 자동화는 제외

처음엔 번역까지 AI에게 맡길까 고민했지만, 번역 품질은 Lokalise와 전문 번역가에게 맡기는 게 맞다고 판단했다.
그래서 이번 파이프라인에서는 딱 기계적인 반복 작업 3가지만 AI와 스크립트에게 맡겼다.

1. 디자인(Figma)에서 텍스트 추출하기
2. 규칙에 맞춰 키 이름 짓기
3. 정해진 포맷으로 업로드하기

## 1. Figma에서 텍스트 추출

텍스트를 뽑기 위해 처음에는 Figma MCP로 화면 노드의 메타데이터를 조회해 보았다. 그런데 상위 "섹션" 노드에는 구조만 있고 실제 텍스트 데이터가 포함되어 있지 않았다. 

그래서 방식을 약간 바꿨다. 개별 프레임들을 스크린샷으로 찍은 다음, 그 이미지 안의 텍스트를 직접 읽어내는 방식으로 진행했다. 숨겨진 프레임이나 공통 헤더는 걸러내고, 여러 화면에 중복되는 텍스트는 하나로 합치도록 했다. 살짝 번거로워 보일 순 있어도 확실하게 텍스트를 누락 없이 가져올 수 있는 방법이었다. 

## 2. UI 텍스트와 데이터 구분하기

추출한 텍스트를 전부 다 번역 키로 만들면 안 된다. 실제 유저 데이터나 서버에서 내려주는 값들은 제외해야 하기 때문이다. 

- **번역 키 생성 대상 (UI 텍스트):** 모달 제목, 버튼, 테이블 헤더, 안내 문구 등
- **제외 대상 (서버값/데이터):** 테이블 셀 내부 데이터(이름, 날짜 등), 서버 상태값, 구체적인 에러 내용 등

조금 애매한 경우는 "146건이 성공적으로 생성되었습니다" 처럼 데이터와 UI 텍스트가 섞여있는 문장이었다. 이런 건 `{count}건이 성공적으로 생성되었습니다` 처럼 변수를 넣은 템플릿 형태로 만들어 처리했다. 

## 3. 키 이름 규칙 적용

텍스트 분류가 끝나면 키 이름을 지어준다. 규칙을 명확하게 문서화해서 항상 일관된 이름이 나오도록 세팅했다. 
어드민 웹의 경우 보통 아래처럼 4단계 점 표기법을 사용한다. 

```text
{도메인}.{UI컨텍스트}.{설명}.{타입}

# 예시
공간관리.모달.공간수정.제목
완료취소내역.필터.조회기준선택.항목
```

이렇게 규칙을 정해두고, 새로운 키를 만들기 전에는 항상 기존 키 목록을 뒤져서 중복되는 내용이 있는지 먼저 확인하도록 했다. 

## 4. 구글 시트 거쳐 Lokalise로 업로드

생성된 키는 바로 Lokalise로 올리지 않고 우선 구글 시트에 CSV 형태로 저장했다. 기획자나 다른 팀원들이 코드를 보지 않고도 시트에서 쉽게 번역 키와 내용을 검토하고 수정할 수 있게 하기 위해서다. 

시트에서 내용이 확정되면 이제 스크립트가 Lokalise로 밀어 넣는다. 

```text
Figma 스크린샷 
  → 텍스트 추출 / 분류 / 키 생성
  → 구글 시트에 CSV 저장 (팀 리뷰)
  → convert.py (시트 데이터를 Lokalise JSON 포맷으로 변환)
  → lokalise2 CLI로 업로드 
  → Process API로 결과 확인 후 Slack 알림 발송 
```

`convert.py`가 시트 내용을 Lokalise용 JSON으로 변환해주고, 로컬 키체인에 저장해둔 토큰을 활용해 CLI로 업로드한다. 
참고로 Lokalise CLI는 비동기 처리라 업로드 결과가 바로 오지 않아서, REST API로 상태를 계속 체크(폴링)하다가 작업이 끝나면 최종 결과를 Slack으로 쏘도록 추가 작업도 해두었다! 

## 마무리 

이렇게 파이프라인을 구축해두니 확실히 다국어 작업이 편해졌다. 예전에는 화면 보면서 일일이 엑셀에 키 이름을 쳐넣어야 했는데, 이제는 Figma 링크만 주면 추출부터 업로드까지 알아서 착착 진행된다. 
앞으로는 디자인이 다 끝난 뒤가 아니라 기획 단계의 프로토타입 코드에서 바로 텍스트를 파싱해오는 방식도 고민 중이다. 아무튼 반복적인 잡일을 자동화해두니 본래의 업무에 더 집중할 수 있어서 만족스럽다!
